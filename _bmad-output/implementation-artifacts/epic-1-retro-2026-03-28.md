## Successes

1. **Vertical Slice Delivery** — Complete product browsing flow end-to-end: PostgreSQL → Elasticsearch sync → REST/GraphQL APIs → Customer Website UI. All 8 FRs delivered in a single epic.

2. **Reusable Foundation Patterns** — BaseEntity, GlobalExceptionHandler, CorrelationIdFilter, JacksonConfig (Story 1.1) reused across all subsequent stories. Outbox Pattern (Story 1.3) established reliable DB→Kafka messaging for the entire project.

3. **Strong Testing Culture** — 200+ tests with proper testing pyramid (unit + integration with Testcontainers). AssertJ for assertions, TestData builders for test data, @IntegrationTest annotation pattern. Zero production incidents.

4. **Effective Code Review Process** — ~26 patches caught and fixed across 7 stories. Security issues (unbounded input, hardcoded URLs), null safety gaps, transaction boundary problems all caught before merge.

5. **Technology Alignment** — Successfully adopted Spring Boot 4.0.4, Jackson 3.x, Spring Framework 7, Vue.js 3.5.x, PrimeVue 4.5.4, Tailwind CSS 4.2.2. Migration challenges documented for team reference.

6. **Observability from Day 1** — Structured JSON logging (logstash-logback-encoder), correlation IDs (X-Correlation-Id header), Micrometer tracing — all established in Story 1.1 and used throughout.

7. **Accelerating Velocity** — Early stories (1.1-1.3) invested heavily in infrastructure and patterns. Later stories (1.4-1.7) benefited from this investment with faster delivery. Story 1.7 completed 11 tasks and 78 tests in a single pass.

## Challenges

1. **Spring Boot 4 Breaking Changes** — Jackson 3.x package rename (`com.fasterxml` → `tools.jackson`), removed APIs (`Jackson2ObjectMapperBuilderCustomizer`, `TestRestTemplate`, `@AutoConfigureMockMvc`), autoconfigure restructuring. Required discovery through trial-and-error as migration documentation was incomplete.

2. **Accumulating Deferred Work** — 21 deferred items across 7 stories, primarily in error handling, resilience, and production hardening. Many deferred to Epic 8 (Resilience). While individually justified, the cumulative volume needs monitoring.

3. **Null Safety Gaps** — Null safety issues appeared in 5 out of 7 stories during code review. Examples: `toLong/toInt` without null checks (1.3), category BatchMapping null (1.5), NaN productId (1.7), unbounded correlation ID (1.1). Systemic issue needing process improvement.

4. **Heavyweight common-lib** — spring-boot-starter-web, -data-jpa, -actuator are compile-scope in common-lib, forcing all consumers to drag in full web server + JPA + actuator. Should split DTOs/exceptions into lighter module when more services are added.

5. **Missing Validation Tests** — No performance test for 500ms p95 SLA (Story 1.4), no relevance ranking test with realistic datasets, HandlerMethodValidationException not handled (Spring Framework 7 validation may return 500 instead of 400).

6. **Hardcoded Values** — API base URLs, price slider maximums, timeouts appeared as hardcoded values in multiple stories. Pattern of not externalizing configuration discovered through review.

---

## Key Insights

1. **Foundation investment pays dividends** — Patterns established in Stories 1.1-1.3 (BaseEntity, GlobalExceptionHandler, Outbox Pattern, design tokens) dramatically accelerated subsequent stories. The vertical slice approach proved valuable.

2. **Code review is an effective safety net, but shift-left is better** — 26 patches found in review means 26 issues that could have been caught earlier. A dev checklist for common issues (null safety, input validation, configurable values) would reduce review burden.

3. **Null safety + input validation is a systemic gap** — Not a one-off issue. Appeared in 71% of stories (5/7). Needs to become a standard part of the development process, not something caught only in review.

4. **Deferred work tracking works** — All 21 items documented with clear context, story origin, and rationale. No lost context. The deferred-work.md file provides good visibility into accumulated technical debt.

5. **Spring Boot 4 migration knowledge is preserved** — Jackson 3.x patterns, autoconfigure restructuring, Testcontainers 2.x changes all documented in MEMORY.md and patterns.md. Future development won't re-discover these issues.

6. **Technology stack is modern and cohesive** — Spring Boot 4 + Vue.js 3 + PrimeVue 4 + Tailwind 4 work well together. No major compatibility issues after initial migration hurdles.
